# KotobaLab Docs

This directory is for project documents that should be useful beyond the current working session.

## Structure

- `product/`: product requirements, MVP scope, user-facing feature boundaries.
- `dictionary/`: dictionary schema, SQLite / SwiftData decisions, dictionary generation pipeline.
- `architecture/`: app architecture notes, module boundaries, dependency flow, testing strategy.
- `roadmap/`: long-term product and technical roadmap.
- `phases/`: implementation phase records, milestone summaries, and acceptance notes.

## Local Notes

Local scratch notes belong in `docs/_local/`.

That directory is ignored by git. Use it for temporary notes, commit-message drafts, AI handoff drafts, or unfinished thinking that should not be uploaded.
