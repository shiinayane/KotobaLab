# Phase Records

Use this directory for milestone records and implementation phase notes.

Suggested naming:

```text
phase-01-database-pipeline.md
phase-02-core-experience.md
phase-03-architecture-tests.md
```

Each phase note should include:

- Goal
- Scope
- Non-goals
- Key decisions
- Verification
- Remaining risks
